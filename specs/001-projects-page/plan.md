# Implementation Plan: Projects Page

**Branch**: `001-projects-page` | **Date**: 2026-04-30 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/001-projects-page/spec.md`

## Summary

Build a fully bilingual, SSR-compatible projects listing and detail page for a personal portfolio site. Projects are stored as static JavaScript data; the listing uses native IntersectionObserver for infinite scroll (6 per batch); detail pages show a gallery, description, objectives, technologies, and optional live/GitHub links.

## Technical Context

**Language/Version**: JavaScript (ES2022), Nuxt 4 / Vue 3  
**Primary Dependencies**: Pinia (`@pinia/nuxt`), `@nuxtjs/i18n`, `@nuxtjs/tailwindcss`, `@nuxtjs/color-mode`  
**Storage**: Static JS data file (`store/projectsData.js`) — no async/API calls  
**Testing**: `@nuxt/test-utils`  
**Target Platform**: Web (SSR, deployed to Vercel)  
**Project Type**: Web application (personal portfolio)  
**Performance Goals**: Grid renders within 2s on initial load; infinite scroll batch visible in < 200ms  
**Constraints**: No new npm dependencies; SSR-compatible (IntersectionObserver guarded in `onMounted`); fully bilingual (EN-US + PT-BR); light + dark theme  
**Scale/Scope**: ≤ 50 projects; 6 per infinite-scroll batch; 2–3 images per project

## Implementation Phases

### Phase A — Data Layer

**Goal**: All project data accessible through the store before touching any UI.

1. Create `store/projectsData.js` with at least 2 sample projects using the data model from `data-model.md`. Each project must have both locale keys in i18n, a thumbnail, at least 1 gallery image, and at least one external link.
2. Create `i18n/projects.js` with `projectsPagePTBR` and `projectsPageENUS`. Include: `heading`, `searchPlaceholder`, `allFilter`, `noResults`, `liveDemo`, `sourceCode`, all `categories.*` keys, and `items.<slug>.*` keys for the sample projects.
3. Register `projects.js` translations in `i18n.config.ts` (add to both locale message objects).
4. Create `store/projects.js` (Pinia store) with:
   - State: `projects` (from projectsData), `currentPage: 1`, `pageSize: 6`, `selectedCategory: null`
   - Getters: `filteredProjects`, `paginatedProjects`, `hasMore`, `categories` (unique list), `getProjectById`
   - Actions: `loadMore()` (increments `currentPage`), `setCategory(key)` (sets `selectedCategory`, resets `currentPage` to 1)
5. Write unit tests in `tests/store/projects.spec.js` covering: `paginatedProjects` slicing, `hasMore` boundary, `getProjectById` hit/miss, `setCategory` resets page.

### Phase B — Routing Migration

**Goal**: Replace Nuxt 2 `_id.vue` with Nuxt 4 `[id].vue` before building new UI.

6. Create `pages/projects/[id].vue` implementing the detail layout from `contracts/components.md`:
   - Load project via `getProjectById(route.params.id)` from the store
   - Render all sections: title, date, category, gallery, description, objectives, technologies, live/github link buttons
   - Translate all display text using `$t(project.titleKey)` etc.
   - Show "not found" state when project is `undefined`
   - External links: `target="_blank" rel="noopener noreferrer"`
7. Delete `pages/projects/_id.vue`.
8. Verify the route `/projects/<sample-id>` works for both locales (`/projects/...` and `/projetos/...`).

### Phase C — Listing Page with Infinite Scroll

**Goal**: Projects grid with filter + infinite loading.

9. Create `components/projects/ProjectCard.vue` — renders thumbnail, translated title, translated category; wraps in `NuxtLink` to `/projects/<id>`; supports dark/light mode.
10. Update `components/projects/ProjectsFilter.vue`:
    - Accept `categories` prop (array of i18n keys) and `modelValue` prop
    - Emit `update:modelValue` on change
    - Render `$t(key)` for each option
11. Update `components/projects/ProjectsGrid.vue`:
    - Use `ProjectCard` for each item
    - Read `paginatedProjects`, `hasMore`, `categories` from store
    - Add `<div ref="trigger" id="infinite-trigger">` sentinel after the card grid
    - In `onMounted`: set up `IntersectionObserver` on sentinel; call `store.loadMore()` on intersection when `hasMore`; disconnect observer when `!hasMore`
    - Pass `categories` to `ProjectsFilter`; on filter change call `store.setCategory()`
    - Show loading skeleton (existing `SkeletonLoader`) while `pending` is true
    - Show `noResults` i18n message when `paginatedProjects.length === 0`
12. Update `pages/projects/index.vue` to ensure it simply mounts `ProjectsGrid` (no local state needed).

### Phase D — Assets

13. Add placeholder/sample project images to `public/images/projects/<slug>/` for each sample project (thumbnail + at least 1 gallery image).
